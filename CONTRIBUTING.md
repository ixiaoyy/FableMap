# Contributing to FableMap

## 目标

FableMap 欢迎对以下方向的贡献：

- OSM 标签解析与语义映射
- 世界 Schema 设计
- 世界记忆与持久化机制
- Godot 场景加载与交互
- 文档、示例数据与开发工具

## 当前贡献优先级

1. 完成 Python CLI 骨架
2. 建立 OSM -> 世界 Schema 的最小映射规则
3. 输出可重复生成的示例 `world.json`
4. 建立 Godot JSON 加载器与基础预制体实例化

## 贡献原则

- **先稳定，再扩展**：优先保证同一区域的生成一致性
- **先 Schema，再渲染**：所有新增功能应优先落到统一数据协议
- **可追溯**：尽量保留 OSM 原始来源与映射依据
- **小步提交**：每次 PR 尽量只解决一个明确问题

## 推荐工作流

1. Fork / clone 仓库
2. 新建功能分支
3. 修改代码或文档
4. 补充示例或测试
5. 提交 PR，并说明：
   - 解决了什么问题
   - 涉及哪些模块
   - 是否影响 Schema 或 Godot 对接

## 提交建议

- 文档改动：说明新增或调整的目标范围
- 规则改动：说明影响了哪些 OSM 标签映射
- Schema 改动：明确兼容性影响
- Godot 改动：说明需要的 JSON 字段与预制体约束

## 当前最需要的贡献类型

- 更多 OSM 标签到幻想语义的映射规则
- 示例区域与对应的输出 JSON
- 世界记忆设计建议
- Godot 占位体和 UI 原型

## 沟通方式

在项目早期，文档一致性与设计清晰度非常重要。

如果你的改动会影响：

- 世界 Schema
- CLI 参数语义
- 存储格式
- Godot 加载协议

请先在 issue 或设计文档中说明，再动手实现。
