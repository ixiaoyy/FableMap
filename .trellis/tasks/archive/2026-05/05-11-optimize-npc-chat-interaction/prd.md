# brainstorm: Optimize NPC chat interaction and mention UX

## Goal

Optimize the NPC interaction flow in the Tavern chat interface to make it more intuitive and less "noisy". Specifically, reduce the initial clutter of multiple NPC greetings and improve the mechanism for mentioning specific NPCs in public chat.

## What I already know

*   **Current State**:
    *   Entering a tavern with multiple NPCs triggers individual "entrance reactions" from all of them, which can clutter the chat log.
    *   Mentioning an NPC in public chat currently requires typing `@` and selecting from a popup, or manual typing.
    *   Clicking an NPC in the `NpcSeatGallery` or sidebar force-switches the user to a private chat channel, even if they are in public mode.
*   **Target Component**: `frontend/app/features/tavern-chat-workbench/index.tsx` (and potentially `TavernChatRoom.jsx` for legacy compatibility).
*   **User Feedback**:
    *   "默认展示所有NPC" (defaults to showing all NPCs) - feels cluttered.
    *   "@点击某一个，才会复制到对话框里面，很奇怪" (clicking @ one copies to dialog box, very strange) - feels manual/unintuitive.

## Assumptions

*   Users often want to "target" an NPC in public chat without switching to a private DM.
*   A single group greeting is better than 5+ individual "hello" messages for a better first impression.

## Open Questions

*   **Preference**: When clicking an NPC avatar in Public Chat, should it:
    1.  Append `@NPCName ` to the textarea (current manual behavior but triggered by click).
    2.  Set a "Targeted NPC" state in the UI that persists for the next message (cleaner, no text prefix needed).
    3.  Open a context menu (Mention / Private Chat / View Detail).

## Requirements (evolving)

*   [ ] **Concise Greetings**: If NPCs > 3, combine entrance reactions into a single group greeting or summary.
*   [ ] **Click-to-Mention**: Clicking an NPC seat/avatar in Public Chat mode should facilitate mentioning them.
*   [ ] **Visual Feedback**: Clearly indicate if a message is being sent to everyone or a specific NPC in public chat.

## Acceptance Criteria

*   [ ] Upon entering a tavern with 5 NPCs, the chat log shows a summarized greeting instead of 5 separate messages.
*   [ ] Clicking an NPC in the sidebar while in Public Chat mode inserts `@NPCName ` into the input and focuses it.
*   [ ] Mentioning an NPC via click feels faster and more natural than manual typing.

## Out of Scope

*   Backend changes to message parsing (keep it compatible with current `@` parsing).
*   Redesigning the entire chat UI theme.

## Technical Notes

*   Main logic in `frontend/app/features/tavern-chat-workbench/index.tsx`.
*   Entrance messages generated by `publicEntranceMessages`.
*   Mention parsing in `parsePublicMentionTarget`.
