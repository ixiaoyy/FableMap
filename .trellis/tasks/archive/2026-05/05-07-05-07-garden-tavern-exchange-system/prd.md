# 05-07-garden-tavern-exchange-system

## Goal
在已完成的菜园酒馆基础种植/浇水/收获流程上，增加“交易所”薄层：让访客能查看作物行情、出售库存作物，并通过农场管家/NPC 文案获得价格变化播报。

## Requirements
- 在菜园酒馆面板中增加交易所/行情区域，展示已有作物的当前价格、涨跌幅和简短趋势说明。
- 复用前端已有菜园状态与库存数据，不新增后端表或跨酒馆经济系统。
- 支持访客把库存作物出售为前端模拟收益，出售后扣减对应库存。
- 使用“西瓜锚定价”作为市场提示的一部分；其它作物价格可用确定性前端模拟，避免随机刷新导致验收不可复现。
- 增加农场管家风格的行情播报文案，但不让平台自动创作酒馆/NPC 正式内容。
- 保持移动端和窄屏布局可用。

## Acceptance Criteria
- [x] 菜园酒馆 UI 能看到交易所/行情入口或卡片。
- [x] 每种已支持作物展示价格、涨跌幅/趋势和简短播报。
- [x] 当库存中有作物时，可以出售单种作物并看到库存减少、收益增加。
- [x] 库存为空或某作物数量为 0 时，出售操作不可用或有明确提示。
- [x] 交易所状态在现有前端持久化边界内保存，不新增数据库/API 合约。
- [x] `npm --prefix .\frontend run build` 通过。

## Technical Notes
- 开发类型：frontend。
- 主要复用 Phase 1 菜园基础任务的前端组件/本地状态。
- 避免实现跨酒馆交易、充值结算、排行榜、真实货币兑换、付费加速。
- 若涉及新增工具函数/常量，先搜索现有 farm/garden/market 相关实现，避免重复。
