# 05-07-garden-tavern-currency-economy

## Goal
在菜园酒馆已有种植、交易所、偷菜基础上，补齐前端 MVP 的“黑钻”经济闭环：货币获取、种子/道具消费、每日补给和酒馆内本地经济看板。

## Requirements
- 保持经济系统为酒馆内轻量副玩法，不接入真实货币、平台充值、Token 结算或跨酒馆交易。
- 黑钻作为前端模拟游戏币：出售作物、偷菜后出售、每日补给可增加余额。
- 种植消耗种子成本；余额不足时种植按钮不可用，并由 NPC 管家提示如何获得黑钻。
- 提供每日登录/进入补给，每日只能领取一次，使用本地 visitor progress 记录。
- 提供农场商店道具薄层，至少支持一次性高级种子包解锁高级作物。
- 提供酒馆内本地经济看板，只展示当前访客的本地统计/自排名，不做跨酒馆/公开排行榜。
- 复用现有 `tavernFarmModes.js` 与 `GardenFarmPanel.jsx`，不新增后端 API/Schema。

## Acceptance Criteria
- [x] 菜园面板显示黑钻余额、每日补给入口和本地经济指标。
- [x] 余额不足时不能种植需要成本的种子，领取补给后可种植低成本作物。
- [x] 种植会扣除对应种子成本。
- [x] 出售作物会增加余额并累计本地收益统计。
- [x] 商店可购买一次性高级种子包，购买后解锁高级种子入口。
- [x] NPC prompt 支持补给、购买、余额不足和酒馆内本地排行播报边界。
- [x] `npm --prefix .\frontend test` 与 `npm --prefix .\frontend run build` 通过。

## Technical Notes
- 开发类型：frontend。
- 持久化边界仍为 localStorage 中的 farm progress；本地身份不是安全授权。
- “排行”仅为当前访客本地统计看板/自排名，不实现公开社交榜、跨酒馆榜或真实访客比较。
