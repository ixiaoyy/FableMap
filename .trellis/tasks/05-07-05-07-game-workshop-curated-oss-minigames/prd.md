# 子任务：游戏工坊 OSS 小游戏白名单（B1/C1 前序）MVP

## Goal

围绕“游戏工坊”提供可直接游玩的老开源小游戏入口，先采用白名单制 + 本地静态托管，先确保可玩且可控，不引入上传与外部不可信源。

## Parent

`D:\work\ai-\.trellis\tasks\05-07-05-07-embedded-story-minigames-game-workshop-brainstorm\`

## 需求范围

- 先做 1–2 个经典轻量游戏（推荐 2048 类）纳入白名单。
- 明确每个游戏的元信息（名称、来源、许可、版本、作者、来源 URL、更新时间、哈希）。
- 提供统一“游戏卡片”入口，点击后在独立视图运行（优先 iframe + sandbox）。
- 游戏加载策略：
  - 首选：游戏资源作为仓库资产内嵌（便于离线和 CSP 收敛）；
  - 次选：受信任 URL + 固定 domain 白名单（若必须外链）。
- 沙箱与通信基线：
  - 使用 `sandbox` 限权（至少不含 `allow-popups`、`allow-top-navigation` 等高危权限）；
  - 通过 `postMessage` 做简单开始/结束/重置事件回传（可选，非强依赖）。
- 与能力卡片对齐：
 - “游戏工坊”在 D1 空间能力卡片中作为独立入口；
 - 完成后返回聊天入口可见。

## 接受标准

- [ ] 游戏入口可打开并启动至少 1 个 OSS 小游戏。
- [ ] 游戏资源来源可审计，许可信息完整记录在任务内（MIT / Apache / etc）。
- [ ] 当前不支持任意上传；只允许白名单内游戏。
- [ ] 无新增数据库字段；若需外链，满足前端资源域收敛策略。
