# FableMap v0.1 工程计划

## 目标

在最短时间内完成“世界生成最小闭环”，并为 Godot Demo 提供稳定输入。

## 第 1 周：CLI 与数据抓取

目标：把坐标变成基础地理数据。

任务：

- 建立 Python 项目结构
- 建立 `generate` 命令入口
- 实现 Overpass API 请求
- 输出原始 OSM 摘要 JSON

交付物：

- 可执行的命令行入口
- 一份原始地理摘要样例

## 第 2 周：语义映射与世界 Schema

目标：把 OSM 数据映射为异世界 JSON。

任务：

- 定义 OSM 实体标准化模型
- 建立首批映射规则
- 生成 `region` / `pois` / `roads` / `state`
- 输出有效的 `world.json`

交付物：

- 一份稳定可读的世界 Schema 样例
- 至少 8 类实体映射

## 第 3 周：世界记忆

目标：让同一区域重复生成时保持稳定。

任务：

- 建立区域 key 方案
- 持久化世界 JSON
- 加入状态版本与访问标记
- 支持缓存读取与刷新生成

交付物：

- 本地缓存目录或 sqlite 数据库
- 重复生成一致性验证

## 第 4 周：Godot 原型接入

目标：把世界 JSON 变成可探索场景。

任务：

- 建立 Godot 工程
- 编写 JSON Loader
- 按 `fantasy_type` 实例化基础预制体
- 实现简单 UI 和点击交互

交付物：

- 可移动 Demo
- POI 描述展示面板

## 最小可演示版本定义

满足以下条件即可对外展示：

- 用户输入坐标
- 系统生成世界 JSON
- 同一地点结果稳定
- Godot 可载入并展示该世界片段

## 当前工程建议

第一阶段先不要过度抽象，不要过早拆成微服务，也不要提前引入复杂数据库。

先把：

> CLI + Schema + 本地记忆 + Godot Loader

做通，就是最重要的事情。
