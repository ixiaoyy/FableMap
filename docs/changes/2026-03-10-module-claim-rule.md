# 2026-03-10 模块认领规则更新说明

## 本次做了什么

- 在 `CONTRIBUTING.md` 中加入“开发前认领要求”
- 在 `docs/AI参与开发协议.md` 中加入“开发前认领规则”
- 在 `README.md` 文档导航中加入模块认领说明模板入口
- 新增 `docs/claims/README.md`，作为统一认领模板

## 为什么要做

为了避免多人同时开发同一个模块、协议或区域内容，提前把“谁在做什么、边界在哪里”记录下来。

这也能减少 AI 在边界不明确时的自由扩写。

## 影响范围

- 协作流程
- AI 参与开发流程
- 文档导航
- 后续模块 / 区域内容认领方式

## 明确没有改什么

- 没有改世界 Schema
- 没有改阵营、扰动或审美系统设计
- 没有引入新的代码实现
- 没有改变现有 Git 远端配置

## 是否涉及协议 / Schema / 命名变更

- 涉及协作协议变更
- 不涉及世界数据 Schema 变更
- 不涉及运行时代码接口变更

## 验证方式

- 检查 `CONTRIBUTING.md` 新增规则是否完整
- 检查 `docs/AI参与开发协议.md` 是否与贡献指南一致
- 检查 `README.md` 导航是否能找到新模板

## 风险点

- 协作流程会比之前多一步，需要后续通过模板保持轻量
