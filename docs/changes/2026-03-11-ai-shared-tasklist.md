# 变更说明：AI 协作共享任务列表

## 本次改动

- 新增 `docs/AI_SHARED_TASKLIST.md`
- 新增对应模块认领说明
- 更新 `README.md` 文档导航
- 更新 `CONTRIBUTING.md` 与 `docs/AI参与开发协议.md`，明确共享任务列表与认领流程的关系

## 改动目的

把当前会话中沉淀的任务，从内部临时任务树整理为仓库内统一共享的任务入口，方便多位开发者与 AI 协作者直接查看可认领项，并按既有认领说明流程开展并行开发。

## 影响范围

- 只影响协作文档、任务入口与导航
- 不修改任何运行时代码或协议字段
- 不替代 `docs/claims/` 与 `docs/changes/` 的既有责任边界

## 验证

- 文档间术语一致性检查
- `git diff --check`

## 备注

共享任务列表负责“当前要做什么”；认领说明负责“谁来做、边界是什么”；变更说明负责“这次实际做了什么”。三者应同时存在，而不是互相替代。
