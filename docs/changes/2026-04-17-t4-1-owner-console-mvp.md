# T4.1 Owner Console MVP — 我的酒馆控制台

## 概述

本次变更完成 Phase 4 的 `T4.1 我的酒馆面板` MVP 收口，把已有店主管理列表整理成可直接使用的控制台入口。

## 为什么改

当前产品主线已经转向“赛博酒馆 UGC 平台”。店主侧需要一个稳定入口，用于查看自己创建的酒馆、管理营业状态、进入编辑和 AI 配置，而不是只停留在地图发现页。

## 改了什么

### `frontend/src/TavernOwnerPanel.jsx`

- 新增店主摘要指标：
  - 全部酒馆数
  - 营业中 / 歇业中数量
  - 角色总数
  - Token 消耗汇总
- 新增列表刷新按钮。
- 新增搜索与筛选：
  - 按酒馆名称、描述、场景、角色名搜索
  - 按营业状态筛选
  - 按访问权限筛选
- 新增筛选空状态与清除筛选动作。
- 增强无 ownerId 时的错误提示。
- 坐标显示增加容错，避免缺失坐标时报错。
- 保留已有创建、编辑、AI 配置、开关店、删除路径。

### `frontend/src/styles.css`

- 新增店主控制台摘要区、筛选区、状态徽章和响应式布局样式。
- 保持现有视觉体系，不新增前端依赖。

### `docs/CURRENT_TASKS.md`

- 将 `T4.1 我的酒馆面板` 标记为 `done`，说明当前完成的是 MVP 控制台能力。

## 没改什么

- 未修改 Tavern / Character / LLMConfig Schema。
- 未新增后端 API。
- 未修改地图发现、酒馆入场、聊天、角色导入协议。
- 未引入新的依赖。

## 影响范围

- 店主视图 `TavernOwnerPanel`
- 前端全局样式 `styles.css`
- 当前任务与协作记录文档

## 验证方式

- `npm run build`（在 `frontend/` 目录执行）通过。

## 风险点

- 当前仍复用前端本地 `visitorId` 作为店主身份，后续如果接入正式账号系统，需要替换 ownerId 来源。
- Token 汇总依赖 tavern payload 中 `llm_config.token_used`，没有记录时显示为空值。
