# T5.5 性能优化

## 背景

T4.4 / T4.5 已经把空间发现、地图 marker 与分类图例接起来。大量空间同时返回时，前端会在筛选、地图 marker 和列表卡片上产生明显压力。

## 改动

- `App.jsx` 为空间发现建立 memoized search index，避免每次筛选都重复拼接搜索文本。
- 地图 marker 限制为最多 80 个，优先显示当前排序/筛选结果，并始终保留当前选中空间。
- 地图状态与图例显示 marker 数量和匹配总数，截断行为对用户可见。
- 空间发现列表改为 12 条一批逐步加载，避免一次性渲染大量卡片。
- 发现列表提示当前地图 marker 数与列表未展开数量。

## 验证

- `npm run build`
- `git diff --check`
