# FableMap Web-2D 灵觉视图设计说明 v0.1

## 目标

这份文档定义 FableMap 在浏览器中的第一个强感知里程碑：基于真实地图骨架，做出一个可切换多种世界镜头的 Web-2D 灵觉视图，而不是只停留在文字卡片或静态 JSON 展示。

## 为什么先做 Web-2D

- 复用当前已经存在的 `nearby -> page -> bundle/index.html` 闭环
- 用标准浏览器能力先证明“现实地图 -> 幻想世界主舞台”成立
- 避开 3D 建模、资源生产和引擎整合的高前置成本
- 为未来 Godot / Pixi / Canvas 版本提供稳定的交互与视觉规格

## 设计原则

### 1. 真实地理骨架保持稳定

无论切到哪条世界线，以下层都不应漂移：

- 道路关系与主要方向
- POI 坐标与相对邻接
- 区域 narrative / faction / state 基础信息
- `world.json` 中既有 `region`、`roads`、`pois`、`landmarks`、`state` 语义

### 2. 风格切换是世界镜头，不是滤镜

可切换的应是整套解释层：

- tiles / 图标 / 建筑装饰
- 色板、光感、空气感
- NPC 样式与密度表达
- 底部电台文案与侧边说明语气
- 可见收集物与情感钩子

### 3. 主地图承载空间，文字退到信息层

主舞台负责让用户“看懂这片区域像一个可行走的异世界”；深度社会解构、情感记忆、任务说明与 ASMR 文本主要进入底栏和侧边栏。

### 4. 现实动态信号优先转译成隐喻代理

浏览器 MVP 不追求完整模拟现实交通或人流，而是把它们转成：

- 拥堵 -> 迷途行者密度上升
- 通畅 -> 异位面商人稀疏游走
- 治愈模式高活性地点 -> 都市精灵刷新增强

## 基准模式

### 模式 A：Dark Satire Spirit Vision

建议以 `gritty_cyberpunk` + `dark_gothic` 为主。

- 道路：冷色石板 / 霓虹能量流 / 工业边缘噪声
- 医院：灵魂收容所、记忆审判庭、白塔式设施
- 便利店：炼金补给站、多巴胺补剂店
- 银行 / 政务设施：堡垒化、教团化、权力结构可视化
- 文案：社会讽刺、制度疲惫、结构性不安

### 模式 B：Healing Spirit Vision

建议以 `ghibli_town` + `impression_garden` 为主。

- 道路：花瓣路径 / 羊皮纸质感 / 柔和风感
- 医院：记忆礼堂、重生教堂、水晶穹顶
- 便利店：都市精灵屋、治愈补剂铺、藤蔓小屋
- 公园 / 水边：精灵刷新点、秘密花园、情绪锚点
- 文案：温柔、抚慰、可分享、偏内心独白

## 现实要素到 2D 实体的最小转义表

- `highway=*` -> 可通行街道骨架；不同模式只改变材质与边缘特效
- `amenity=hospital` -> 高辨识度核心建筑；可被解释为医疗权力设施或情感重生空间
- `shop=convenience` -> 新手高频交互点；可承载补给、收集、留言、兑换
- `amenity=bank` / `office=*` -> 权力与资本节点；适合暗黑模式强化
- `leisure=park` / `natural=*` -> 治愈模式中的精灵、记忆与停留点
- `building:levels` -> 建筑体量、塔楼高度、压迫感或庇护感
- `traffic` / 拥堵信号 -> NPC 密度、街道活性、异常压力

## 分层渲染建议

### v0.1：SVG / DOM 快速闭环

- 道路、建筑、POI 用 SVG 画出空间关系
- 用 CSS 变量切换色板与 UI 气质
- 用简化图标 / 像素块模拟建筑语义
- 完成点选、悬停、侧边详情与底部情绪电台

### v0.2：Canvas / Pixi 升级层

- 把道路流光、建筑呼吸感、漂浮符号移到 Canvas 层
- 增加简单位移动画、NPC 行走、粒子效果
- 保持 `world.json` 与模式配置稳定，不重写世界协议

### v1：HD-2D 表现层

- 局部 shader、光晕、雨夜反光、雾层、花瓣或电子噪声
- 建筑 prefab 进一步语义化
- 缩放时触发“现实剥落 -> 灵觉覆盖”的过渡

## UI 架构

- 顶部：地点、模式、语言、缩放与切换控件
- 左/右侧边栏：建筑解构、任务、阵营、记忆、私密叙事
- 底部情绪电台：`state.active_lens`、区域 ASMR、异常压力、浪漫因子等
- 主舞台：道路、建筑、精灵 / NPC、可点击实体

## 当前建议的 MVP 顺序

1. 在现有 `bundle/index.html` 中确立地图主舞台，而不是继续堆卡片
2. 做稳定的“共享骨架 + 双模式切换”
3. 为医院、便利店、银行、公园、道路做首批语义图标
4. 增加侧边详情与底部电台
5. 再引入 NPC 密度与精灵刷新等动态代理层

## 明确不在 v0.1 的目标

- 完整 GIS 瓦片系统
- 高精度导航与路径规划
- 真实多人同步
- 直接跳到 3D 或重型引擎重做
