# FableMap 玩家状态系统设计 v0.1

## 目标

这份文档定义浏览器 2D 主舞台与后续世界系统共用的玩家状态设计。

目标不是把玩家做成传统 RPG 的 `HP / MP / EXP` 面板，而是把玩家状态定义为：**玩家在这座城市异世界里，正处于什么样的存在方式**。

它需要同时回答五个问题：

1. 玩家现在能不能行动
2. 玩家现在会看到什么世界
3. 世界会如何解释玩家
4. 玩家在这片区域留下了什么痕迹
5. 玩家长期更像什么样的人

因此，玩家状态系统应服务以下方向：

- 当前 Web-2D 地图主舞台中的移动、观察、交互与 UI 展示
- 后续阵营、事件、家园、精灵、记忆与扰动系统
- 长期的审美调谐、人格画像与城市共创体验

## 设计总纲

玩家状态变化，不应只是属性增长，而应是**玩家与城市互相塑形的过程**。

也就是说：

- 你走过哪些路，决定你更能看见什么
- 你常停留在哪些地方，决定哪些地方开始记住你
- 你帮助了谁、忽视了谁，决定城市如何命名你
- 你偏爱哪种风格，决定现实被翻译成怎样的第二世界

## 三层结构

玩家状态建议拆成三层：

1. **即时状态层**：秒级变化，直接驱动当前地图体验
2. **世界关系层**：中频变化，定义玩家与区域、阵营、地点之间的关系
3. **长期人格 / 审美层**：低频变化，定义玩家最终会变成什么样的人

这样的拆分可以同时兼容：

- 当前右侧“玩家状态”卡片
- 地图观察 / 交互 / 相机跟随
- 区域熟悉度与世界记忆
- 阵营反馈与共创痕迹
- 家园、审美、记忆与长期成长系统

## 一、即时状态层

即时状态层适合优先落地在当前地图主舞台与右侧状态卡中。

特点：

- 高频变化
- 可视化强
- 直接影响移动、交互、地图反馈与文案提示

### 1. 位置状态

位置不应只是一组坐标，而应是“语义位置”。

建议字段：

- `current_position`：当前坐标
- `current_zone_id`：当前所在区域
- `nearest_poi_id`：最近兴趣点
- `road_context`：所处道路语境，如 `arterial / street / pedestrian / edge`
- `terrain_feel`：所处地表与功能气质，如 `commercial / residential / transit / medical / academic / park`

用途：

- 决定右侧面板当前展示的地理语境
- 决定广播、NPC、提示语气
- 决定是否触发地点详情、进入边界、区域切换等事件

### 2. 行动状态

当前系统已有“驻足”概念，建议扩展为小型状态机，而不是单一文本字段。

建议状态：

- `idle`：驻足
- `moving`：行进
- `observing`：观察
- `interacting`：交互中
- `collecting`：收集中
- `listening`：聆听广播 / 回声
- `tracing`：追踪线索
- `returning_home`：回家园中

用途：

- 决定相机是否自动跟随
- 决定附近可交互点是否高亮
- 决定右侧卡片展示是偏信息流还是偏沉浸视角
- 决定地点细节何时展开、广播何时压低优先级

### 3. 感知状态

感知状态比传统 RPG 的生命值更重要。它不是失败惩罚，而是世界滤镜控制器。

建议最小化为 3 到 5 个轻量指标：

- `clarity`：清明度，影响世界是否清晰、有序
- `tension`：紧张度，影响危险感、异变提示、广播噪声
- `attunement`：感应度，影响隐藏线索、精灵、回声的可见性
- `fatigue`：疲惫度，影响探索效率、UI 信息密度与叙事语气
- `warmth`：情绪温度，影响治愈内容、家园反馈与收藏触发

建议语义：

- `clarity` 高：道路更稳定、命名更明确、地图解释更理性
- `tension` 高：异常压力更突出、敌对阵营信息更靠前、文案更不安
- `attunement` 高：隐藏节点、都市精灵、记忆裂缝更容易显现
- `fatigue` 高：建议返家、休整或减少信息曝光
- `warmth` 高：更容易触发温柔型叙事与家园共鸣

### 4. 即时状态与当前 D2 地图逻辑的关系

对现有地图主舞台，建议采用以下更新链路：

1. 输入产生移动意图
2. 系统尝试更新玩家位置
3. 道路骨架决定本次位移是否成立
4. 若位移成立，则更新相机与位置语义
5. UI 回写最终 `action_state` 与位置信息
6. 感知状态根据区域与事件做轻量波动

也就是说，玩家状态应由“**是否发生有效位移、位于什么语境、正在做什么**”共同决定，而不只是由按键本身决定。

## 二、世界关系层

这一层不是“我是谁”，而是“这片地方怎么看我”。

变化频率中等，适合在一次出行或一段探索过程中渐进变化。

### 1. 阵营关系

阵营状态不建议只做全局好感度，而建议按阵营拆分多个维度。

建议字段结构：

- `faction_standing.<faction_id>.favor`：支持度 / 友好度
- `faction_standing.<faction_id>.trust`：信任度
- `faction_standing.<faction_id>.heat`：警惕度 / 注意度
- `faction_standing.<faction_id>.debt`：欠账 / 未偿还关系

示例：

- 贸易公会对你 `favor` 高，但 `debt` 也高
- 秩序机构对你 `trust` 低，但 `heat` 很高
- 治愈阵营只在特定时间段回应高 `warmth` 玩家

用途：

- 决定 POI 是否开放特定交互
- 决定阵营广播、路人语气、隐藏内容与委托可见性
- 决定城市如何命名与识别玩家

### 2. 区域熟悉度

区域熟悉度是地图体验的核心状态之一。

建议字段：

- `zone_familiarity.<zone_id>`
- `poi_familiarity.<poi_id>`
- `route_familiarity.<route_id>`

用途：

- 控制地图去雾程度
- 控制 POI 名称是否从模糊描述升级为正式名称
- 解锁区域背景、地点历史与局部叙事
- 形成“这条街记住你了”的长期反馈

### 3. 地点痕迹 / 回声

每个区域或地点应能记录玩家留下的行为印记。

建议字段：

- `left_messages_count`
- `restored_sites`
- `disturbances_triggered`
- `memories_bound`
- `spirits_seen`
- `public_actions_taken`

用途：

- 改变某街区对玩家的语气与反馈
- 让某个 POI 成为个人锚点
- 让地点广播引用玩家行为留下的句子
- 为后续共创与世界写回系统提供依据

## 三、长期人格 / 审美层

这一层是长期存档与身份形成的核心。

不是升级，而是玩家逐渐长成什么样的人。

### 1. 漫游者画像

不建议定义固定职业，而建议跟踪长期行为偏好，生成动态 archetype。

示例倾向：

- 偏探索：`wanderer` / `boundary_scout`
- 偏社交：`mediator` / `messenger`
- 偏黑暗线：`rift_archivist` / `night_patroller`
- 偏治愈线：`echo_gatherer` / `greenhouse_keeper`
- 偏审美线：`style_cataloguer` / `light_sampler`

建议统计维度：

- 探索率
- 停留时长
- 收集偏好
- 阵营倾向
- 修复 vs 扰动
- 公共表达 vs 私密收藏

输出形式：

- 动态头衔
- 玩家名片文案
- 角色在世界中的解释方式

### 2. 审美调谐

审美调谐不是换皮，而是让玩家把现实调到自己能居住的频率。

建议字段：

- `style_affinity.ghibli`
- `style_affinity.wongkarwai`
- `style_affinity.cyber_noir`
- `style_affinity.impressionist`
- `style_affinity.warm_corner`

用途：

- 影响地图配色与光感
- 影响地点命名偏向
- 影响回声文案风格
- 影响可收集对象类型
- 影响家园可装配内容与精灵生成倾向

### 3. 私密记忆状态

私密记忆是治愈线与表达线的重要核心。

建议字段：

- `anchored_places`
- `memory_capsules`
- `private_routes`
- `ritual_sites`
- `favorite_weather_moods`

地点可被玩家标记为：

- 安全点
- 雨夜点
- 失眠点
- 恢复点
- 纪念点
- 仅自己可见的点

用途：

- 影响家园陈列内容
- 影响再次到访时的专属文本
- 影响私密广播与个人地图图层

## 四、状态变化触发器

玩家状态不要只靠按钮点击变化，而应由位置、停留、路径、时间与行为类型共同驱动。

建议分为四类触发器。

### 1. 空间触发

当玩家：

- 进入新区域
- 靠近某类 POI
- 穿过阵营边界
- 从主干道转入小巷
- 在高密度地点集群中移动

可改变：

- 感知状态
- 当前语境
- 可见广播
- 阵营热度

### 2. 时间触发

当玩家：

- 停留 10 秒以上
- 在同一地点重复访问 3 次以上
- 在夜晚到访
- 在雨天到访
- 连续多天经过同一区域

可改变：

- 熟悉度
- 地点回声
- 私密记忆
- 精灵 / 线索可见性

### 3. 行为触发

当玩家：

- 观察而不进入
- 进入并交互
- 收集物件
- 绑定记忆
- 留下公共痕迹
- 解决或放任扰动

可改变：

- 玩家画像
- 阵营关系
- 地点状态
- 家园资源
- 世界叙事分支

### 4. 世界触发

未来接入动态扰动系统后，玩家状态也应被世界反向影响。

例如：

- 商圈异常上涨 -> `tension` 升高
- 医疗区进入过载 -> `clarity` 下降，广播更沉重
- 公园处于平静时段 -> `warmth` 与恢复效率提高
- 深夜交通异常 -> 夜间阵营活跃，隐藏节点显现

这样玩家状态不只是“我操作了什么”，也是“城市正在如何作用于我”。

## 五、状态变化节奏

为避免系统噪声过大，建议遵循三档节奏：

### 1. 高频变化

适合秒级刷新：

- `current_position`
- `current_zone_id`
- `nearest_poi_id`
- `road_context`
- `action_state`

### 2. 中频变化

适合一次探索或几分钟内变化：

- `clarity`
- `tension`
- `attunement`
- `fatigue`
- `warmth`
- `zone_familiarity`
- `poi_familiarity`
- `route_familiarity`

### 3. 低频变化

适合多次探索后逐渐累积：

- `faction_standing`
- `style_affinity`
- 玩家画像 / archetype
- 私密记忆集合
- 家园积累与长期痕迹

## 六、MVP 最小状态模型

第一版不应做得过重，建议先落地以下 8 个核心字段：

```json
{
  "position": {
    "x": 0,
    "y": 0,
    "zone_id": "central_block"
  },
  "action_state": "idle",
  "focus_target_id": "poi_pocket_mart_outpost",
  "clarity": 72,
  "tension": 28,
  "attunement": 41,
  "zone_familiarity": {
    "central_block": 18
  },
  "faction_standing": {
    "trade_guild": {
      "favor": 12,
      "heat": 4
    }
  },
  "home_style": "warm_corner"
}
```

这套最小模型已经足以支持：

- 右侧玩家状态卡
- 地点详情联动
- 基础阵营反馈
- 感知条变化
- 镜像家园入口

### MVP 字段说明

| 字段 | 类型 | 含义 | 当前用途 |
| --- | --- | --- | --- |
| `position.x / position.y` | number | 玩家当前位置 | 驱动地图实体、相机与 UI |
| `position.zone_id` | string | 当前所在区域 | 驱动区域熟悉度与语境 |
| `action_state` | enum | 当前行动状态 | 决定 UI 模式与交互反馈 |
| `focus_target_id` | string/null | 当前关注对象 | 联动右侧地点详情 |
| `clarity` | number | 清明度 | 影响文本清晰度与地图解释 |
| `tension` | number | 紧张度 | 影响异常、广播与风险暗示 |
| `attunement` | number | 感应度 | 影响隐藏节点与回声可见性 |
| `zone_familiarity` | map | 区域熟悉度 | 影响名称、去雾与地点解释 |
| `faction_standing` | map | 阵营关系 | 影响阵营反馈与开放度 |
| `home_style` | string | 当前家园主风格 | 联动家园面板与治愈线 |

## 七、右侧“玩家状态”卡片建议

当前卡片若只显示坐标与“驻足”，更像调试输出。

建议升级为世界内状态卡。

### 建议展示结构

- 玩家状态
- 当前位置：`Central Cross / 0,0`
- 行动：`驻足`
- 感知：`清明 72 / 紧张 28 / 感应 41`
- 区域熟悉：`中央街区 Lv.1`
- 当前关注：`Pocket Mart Outpost`
- 阵营回响：`trade_guild 对你保持开放`
- 家园风格：`Warm Corner`

### 展示原则

- 顶部显示即时状态
- 中部显示当前感知与区域关系
- 底部显示长期连接项，如家园风格或当前画像标签
- 避免堆叠过多纯数字，优先展示“世界内解释”

## 八、与现有系统的接口关系

### 1. 与 Web-2D 地图主舞台的关系

玩家状态为地图层提供：

- 玩家实体坐标
- 当前行动状态
- 当前关注对象
- 语义位置解释
- 相机跟随与 UI 联动依据

### 2. 与阵营系统的关系

玩家状态为阵营系统提供：

- 当前阵营 standing
- 玩家长期倾向
- 公开行为痕迹
- 当前区域语境

### 3. 与家园系统的关系

玩家状态为家园系统提供：

- `home_style`
- 私密记忆锚点
- 收集偏好
- 恢复需求与温度倾向

### 4. 与精灵 / 回声 / 记忆系统的关系

玩家状态为这些系统提供：

- `attunement`
- `warmth`
- 熟悉度
- 私密记忆与公共痕迹

### 5. 与动态扰动系统的关系

动态扰动系统反向影响：

- `clarity`
- `tension`
- 可见广播
- 特定节点开放条件

---

## 九、实现建议顺序

### 第一阶段

优先落地：

- `action_state`
- `focus_target_id`
- `clarity / tension / attunement`
- `zone_familiarity`
- `faction_standing`
- `home_style`

### 第二阶段

继续扩展：

- 玩家画像
- 记忆锚点
- 公共痕迹
- 动态扰动的反作用
- 审美调谐树

### 第三阶段

进入长期系统：

- 私密路线
- 个性化城市广播回应
- 家园物件由现实探索反哺
- 同一区域的历史回声累积

---

## 十、结论

FableMap 的玩家状态系统，核心不应是“升级”，而应是：

**玩家如何被城市改变，又如何反过来改变城市。**

因此，玩家状态变化应被视为：

- 感知的变化
- 行动方式的变化
- 世界关系的变化
- 长期人格与审美取向的形成
- 玩家在城市中留下的痕迹不断沉积

这也是玩家从游客转变为共创者的关键接口。
