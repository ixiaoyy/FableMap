# FableMap 产品概述

## 一句话定义

FableMap 是一个把真实地点转化为**可进入的角色场景、地点事件与持续对话体验**的开源叙事系统。

它不再以“自动生成一张异世界地图”为核心目标，而是以**真实底图 + 地点叙事 + 角色遭遇 + 世界记忆**为主线。

## 为什么做

现有产品往往停留在两个极端：

- 传统地图产品有空间结构，但没有角色连续性、故事张力和记忆沉淀
- AI 对话产品有表达能力，但缺少真实地点锚定、空间上下文和回访连续性

FableMap 的目标是把两者连接起来：

- 让 AI 不只是聊天，而是**基于真实地点、真实街区和真实路径聊天**
- 让现实世界不只是地图，而是**可被进入、可被遭遇、可被记住的叙事空间**

## 它最终想成为什么

FableMap 最终想成为的，不是一个“地图换皮”工具，也不是一个“AI 自动画图”的演示产品。

它更像一个覆盖在真实城市上的**地点叙事层**：

- **你打开它时，首先看到的是熟悉的真实底图**：街道、路口、商店、学校、公园、医院仍然是现实中的位置。
- **但你进入某个地点后，看到的不再只是 POI 信息**：而是角色、事件、对话、关系、地点记忆和回访线索。
- **玩家不是游客，而是参与者**：你会留下观察、驻足、标记、聊天选择、事件结果与私密或公共痕迹。
- **城市会逐渐记住发生过什么**：同一个地点会有历史、有状态、有关系变化，有你和其他玩家留下的持续影响。

换句话说，FableMap 希望把“地图”降级为入口，把“地点中的连续故事体验”升级为产品本体。

## 核心价值

### 1. 赋予地点连续叙事能力

现实中的地点不再只是被点击一次就结束的信息点，而会成为：

- 有角色驻留的场景
- 有状态变化的事件节点
- 有前情与后果的回访对象

### 2. 赋予 AI 真实空间上下文

AI 的输出不再漂浮于抽象对话，而是绑定到：

- 真实坐标
- 周边 POI
- 路网与街区关系
- 地点历史与玩家痕迹
- 当前事件与角色关系

### 3. 建立可演化的地点记忆

同一个地点不会每次都随机刷新，而会保留：

- 最近发生的事件
- 玩家曾经做过的选择
- 对角色关系造成的变化
- 地点正在积累的公共或私密叙事

### 4. 让地图成为故事入口，而不是产品负担

FableMap 不再试图自己解决复杂的地图绘制、美术风格统一和场景自动布局问题。

真实地图 SDK 负责：

- 定位
- 空间认知
- 拖拽与缩放
- 地点浏览

FableMap 自己负责：

- 地点体验
- 角色遭遇
- 事件推进
- 对话与记忆

### 5. 让现实世界成为可体验的城市酒馆

用户打开产品后，不是为了看一张更炫的地图，而是为了：

- 看附近哪里有故事
- 看谁正在等你
- 看哪些地点有事件在酝酿
- 进入某个地点展开一次遭遇、一次对话、一次记忆写回

## 目标用户

### 开发者 / 创作者

- 希望快速体验“输入坐标 -> 进入地点 -> 触发角色事件”的技术演示
- 想参与 AI + LBS + 叙事系统融合项目的开源贡献者
- 对地点驱动剧情、持续世界、角色记忆系统感兴趣的开发者

### 玩家 / 体验者

- 对“现实地点变成可进入故事场景”有强烈兴趣的人
- 喜欢角色互动、地点叙事、都市传闻、轻跑团或酒馆式体验的人
- 喜欢黑暗叙事、现实隐喻、城市神秘感与连续对话的人
- 喜欢在熟悉街区中发现隐藏故事、记忆与情绪入口的人

## 目标体验

### 场景 A：桌面城市酒馆模式

用户输入现实位置后，系统在真实底图上展示附近地点。用户点开某个地点后，不是只看到描述卡片，而是进入一个地点面板：这里可能有一个角色、一场对话、一个等待推进的事件，或者一个与过往行为相关的回访反馈。

### 场景 B：通勤中的地点遭遇

用户在熟悉的街区移动时，系统根据地点语义、玩家状态与近期历史，提示附近可进入的角色场景：某个便利店现在像一个补给酒馆，某个路口正在酝酿冲突，某个长椅上留下了一段只对你可见的回声。

### 场景 C：成人向现实寓言体验

用户经过写字楼、健身房、便利店和高消费街区时，系统不只是做奇幻换皮，而是把这些地点解释为权力关系、资源博弈、压力结构与人际张力的入口，并通过角色与事件把现实压力组织成可玩的叙事结构。

### 场景 D：私密回访与情绪留痕

用户再次回到某个地点时，系统能根据上次的对话、事件与 writeback 呈现新的状态：一个角色记得你曾经说过的话，一个地点因为你的标记积累出了新的传闻，一段原本私密的情绪痕迹正在变成后续故事的引子。

## 为什么它可能有传播力

- **低门槛**：任何人都可以从自己熟悉的真实地点进入体验
- **强代入**：地点是自己的，故事却是活的
- **内容可持续**：角色、事件、记忆和回访天然形成连续内容
- **展示性强**：非常适合 demo 视频、地点故事展示和社区传播
- **AI 优势明确**：AI 用于角色扮演、事件组织和连续记忆，而不是硬做地图绘制

## 当前产品策略

当前阶段不再追求“画出一张完整异世界地图”，而是先验证下面这条闭环：

> 现实地点 -> 真实底图入口 -> 地点选择 -> 角色 / 事件 -> 聊天推进 -> writeback -> 回访反馈

只要这个闭环成立，FableMap 就具备继续扩展为平台型项目的基础。

在这个基础上，中长期会进一步扩展为一个同时覆盖“地点遭遇”“角色关系”“公共事件”“城市记忆”的叙事系统，重点引入：

- 地点事件编排
- 角色长期记忆
- 公共广播与区域传闻
- 玩家与地点的关系演化
- 基于真实城市信号驱动的动态事件
- 更成熟的多角色、多地点故事线程

## 当前取舍

当前明确降低优先级的方向：

- 自绘地图主舞台
- 复杂地图视觉资源包体系
- 生成式场景画面
- “地图是否看起来像 RPG 城镇图”这一类审美目标

当前明确抬升优先级的方向：

- 真实底图接入
- 地点面板体验
- 角色引擎
- 事件引擎
- 聊天主舞台
- writeback 与世界记忆

## 一句话总结

FableMap 现在最应该成为的，不是“会自动画地图的 AI 项目”，而是一个**以真实地点为入口、以角色和事件为核心、以聊天和记忆为主舞台的城市叙事系统**。
