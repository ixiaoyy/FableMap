# FableMap 开发路线图

## 阶段 0：文档初始化

目标：统一项目愿景、范围、架构和数据协议。

交付物：

- README
- PRD
- 架构设计
- 世界 Schema
- 开发路线图

## 阶段 1：CLI 世界生成器

目标：输入坐标，输出世界 JSON。

任务：

- 建立 Python 项目结构
- 实现 Overpass API 请求模块
- 提取 OSM 关键标签
- 建立第一版映射规则
- 输出标准世界 JSON

交付标准：

- 可以对指定坐标生成有效 JSON
- 至少支持 8 类常见地理实体映射

## 阶段 2：世界记忆

目标：让同一地点生成结果保持一致。

任务：

- 定义区域 key 方案（H3 或规则网格）
- 建立本地持久化结构
- 加入区域状态字段
- 支持重复生成时复用已有记录

交付标准：

- 同一区域二次生成后核心结构不漂移
- 能记录基础访问状态

## 阶段 3：Godot Demo

目标：把 JSON 变成可探索场景。

任务：

- 建立 Godot 工程
- 编写 JSON 加载器
- 按 `fantasy_type` 生成基础预制体
- 实现玩家移动和点击描述面板

交付标准：

- WASD 可移动
- 至少支持建筑、道路、森林三类表现
- 点击 POI 后可查看描述文本

## 阶段 4：叙事增强

目标：让地图不仅可看，还能承载故事。

任务：

- 为 POI 增加任务钩子
- 增加区域级 faction / event 字段
- 为状态演化预留任务传播接口

## 阶段 5：移动与 AR 探索（后续）

目标：把现实步行体验接入系统。

任务：

- GPS 区域切换
- 轻量事件推送
- 摄像头叠加实验
- 现实地点与游戏内奖励机制

## 当前推荐的短期执行顺序

1. 完成 Python CLI 骨架
2. 完成 OSM -> Schema 映射最小闭环
3. 完成本地世界记忆
4. 输出 2~3 个示例区域 JSON
5. 再开始 Godot 原型
