# FableMap 开发路线图

## 阶段 0：文档初始化

目标：统一项目愿景、范围、架构和数据协议。

交付物：

- README
- PRD
- 架构设计
- 世界 Schema
- 开发路线图

## 阶段 1：CLI 世界生成器

目标：输入坐标，输出世界 JSON。

任务：

- 建立 Python 项目结构
- 实现 Overpass API 请求模块
- 提取 OSM 关键标签
- 建立第一版映射规则
- 输出标准世界 JSON

交付标准：

- 可以对指定坐标生成有效 JSON
- 至少支持 8 类常见地理实体映射

## 阶段 2：世界记忆

目标：让同一地点生成结果保持一致。

任务：

- 定义区域 key 方案（H3 或规则网格）
- 建立本地持久化结构
- 加入区域状态字段
- 支持重复生成时复用已有记录

交付标准：

- 同一区域二次生成后核心结构不漂移
- 能记录基础访问状态

## 阶段 3：Godot Demo

目标：把 JSON 变成可探索场景。

任务：

- 建立 Godot 工程
- 编写 JSON 加载器
- 按 `fantasy_type` 生成基础预制体
- 实现玩家移动和点击描述面板

交付标准：

- WASD 可移动
- 至少支持建筑、道路、森林三类表现
- 点击 POI 后可查看描述文本

## 阶段 4：叙事增强

目标：让地图不仅可看，还能承载故事。

任务：

- 为 POI 增加任务钩子
- 增加区域级 faction / event 字段
- 为状态演化预留任务传播接口

交付标准：

- 地点可以挂载任务和事件入口
- 区域具备基础 faction / event 描述能力

## 阶段 5：成人向叙事系统

目标：让世界具备黑暗、复杂、社会化的成年体验。

任务：

- 为地点建立黑色幽默与社会讽刺模板层
- 让办公楼、商业区、健身房等现实地点拥有更尖锐的异世界映射
- 建立地区阶层感、资源压力与区域气质规则
- 明确暗黑哥特 / 脏冷赛博两类主视觉方向

交付标准：

- 输出一组明显区别于“儿童奇幻”的世界风格样例
- 核心城市设施具备更成熟的叙事解释方式

## 阶段 6：阵营、经济与权力博弈

目标：让地点不只是景观，而是权力网络中的节点。

任务：

- 引入阵营、声望和控制力字段
- 建立贸易路径、资源流和地区价值模型
- 探索现实消费水平、房价、商圈密度等数据的转译方式
- 设计玩家通勤行为如何影响阵营状态

交付标准：

- 区域之间存在可见的阵营差异
- 玩家行为可改变至少一种地区状态

## 阶段 7：隐秘历史与动态扰动

目标：让城市具备历史回声和实时异常。

任务：

- 把历史事件、地点背景和城市传闻转成可检索的 lore 片段
- 设计“历史回声”“异常残留”“谜团线索”等机制
- 把拥堵、人流、营业时间、天气等信号映射成地区扰动
- 产出社会动荡程度等区域动态指标

交付标准：

- 玩家可以追踪至少一条基于真实背景的谜团链
- 世界会因现实动态信号而出现可解释的变化

## 阶段 8：移动与 AR 探索（后续）

目标：把现实步行体验接入系统。

任务：

- GPS 区域切换
- 轻量事件推送
- 摄像头叠加实验
- 现实地点与游戏内奖励机制

## 当前推荐的短期执行顺序

1. 完成 Python CLI 骨架
2. 完成 OSM -> Schema 映射最小闭环
3. 完成本地世界记忆
4. 输出 2~3 个示例区域 JSON
5. 再开始 Godot 原型
