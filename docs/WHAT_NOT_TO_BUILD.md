# FableMap 明确不做清单 v1.4

## 文档目的

这份文档是 FableMap 的**负面清单**，明确列出我们**不应该做的事情**。

---

## 核心原则

> **不是所有 AI 能做的事情，FableMap 都应该做。**

我们的价值来自：
- 主人主权 > 平台控制
- Token 自主 > 平台计费
- 真实锚点 > 无限自由生成
- SillyTavern 兼容 > 平台锁定

---

## ❌ 明确不做的技术方向

### 1. 平台绕过店主确认生成 / 发布空间内容

**不做**：
- 平台自动生成并上线 NPC 角色
- 平台自动生成并发布空间描述、场景、世界书或故事
- AI 未经店主确认就覆盖、改写或扩写既有空间内容
- AI 替店主承担最终创作责任

**原因**：
- 违背"主人主权"核心价值观
- 空间的灵魂在于创作者的独特性
- 未经确认的自动发布会让平台从创作辅助变成内容发布方

**允许的受控替代方案**：
- 店主手动配置角色、描述和设定
- 提供 SillyTavern 角色卡导入降低创作门槛
- 提供 `AI 草稿`：平台可按规则生成未发布 NPC 草稿，但店主必须审核、编辑并确认后才能保存为 TavernCharacter 或发布到空间
- AI 草稿在确认前不得进入公开 Tavern payload，不得覆盖已有 NPC，不得随空间包导出
- 平台只做托管、AI 对话、访问控制和创作辅助，不替店主最终发布内容

---

### 2. 平台级 Token 付费/充值系统

**不做**：
- 平台内置 Token 购买与充值
- 平台与店主进行 Token 结算
- 平台级别的 Token 市场
- 平台抽成 Token 消耗

**原因**：
- 过于复杂，先不做
- Token 由店主自行在 LLM 官网充值即可
- 最小平台原则：平台只做地图、注册、搜索、入口

**替代方案**：
- 店主自行承担 LLM API 费用
- 平台仅记录 Token 使用量供店主参考
- Token 消耗对访客不可见

---

### 3. 无锚点的自由生成空间

**不做**：
- 脱离真实地图的完全虚构空间位置
- AI 自由创造空间坐标
- 无真实坐标约束的漂浮世界

**原因**：
- 失去真实地理骨架的护城河
- 无法与现实城市形成映射关系
- 玩家无法在现实中回访虚拟空间所在的位置

**替代方案**：
- 所有空间必须挂接真实地理坐标
- AI 负责解释现实地点，不负责创造空间位置

---

### 4. 无边界访客社交

**不做**：
- 访客添加好友
- 访客动态墙
- 跨空间访客间消息传递
- 全局在线状态、陌生人匹配、私信收件箱
- 绕过空间主人审批的真人扮演

**原因**：
- FableMap 的核心是"人与空间 / NPC 的关系"，不是通用社交网络
- 社交网络会稀释"空间"的独特性
- 与成熟社交产品竞争没有优势
- 真人参与需要明确的空间范围、角色范围和主人治理

**替代方案**：
- 允许店主开启空间内 `player-as-NPC` 混合扮演模式
- 访客只能申请认领已有 NPC，且需要店主审批 / 拒绝 / 撤销
- 扮演关系绑定单个空间和单个角色，不形成跨空间社交图谱
- 实时聊天、审核策略、在线状态等能力必须另开 Trellis task 做需求评审

---

### 5. 端到端 AI 生成（无结构化中间层）

**不做**：
- 直接让 LLM 生成最终 UI
- 跳过结构化 schema，直接生成散文
- 只能一次性消费的 prompt 幻觉
- 无法落库、无法缓存、无法回放的 AI 输出

**原因**：
- 对话历史不可持久化
- 无法形成长期记忆
- 无法支持多 NPC 上下文

**替代方案**：
- 对话必须结构化落库（ChatMessage schema）
- NPC 记忆必须绑定访客状态
- 所有 AI 输出必须可落库、可缓存

---

### 6. 传统 App 功能

**不做**：
- 不需要 AI 参与也能做好的功能
- 路线规划、POI 搜索、评分系统
- 传统地图 App 的标准功能
- 战斗系统、等级装备系统

**原因**：
- 偏离空间的核心定位
- 与成熟产品竞争没有优势
- 稀释平台的独特性

**替代方案**：
- 只做"AI 参与才能做好"的功能
- 传统功能必须经过空间重新诠释

---

## ❌ 明确不做的产品方向

### 7. 追求"更像游戏"

**不做**：
- 添加战斗系统
- 添加等级装备
- 添加竞技排行榜
- 添加传统 RPG 任务

**原因**：
- FableMap 是空间 UGC 平台，不是游戏
- 游戏化会稀释空间体验的独特性
- 与成熟游戏竞争没有优势

---

### 8. 追求"更像传统地图"

**不做**：
- 添加路线规划
- 添加实时导航
- 添加 POI 评分
- 添加商家入驻

**原因**：
- 与 Google Maps / 高德地图竞争没有优势
- 偏离"空间"的核心定位

---

### 9. 追求"更炫的地图表现"

**不做**：
- 继续增强自绘地图渲染
- 新增地图视觉资源包
- 为了"更像 RPG 城镇图"而引入复杂美术资源
- 让 AI 直接负责生成最终地图画面

**原因**：
- 地图只是空间入口，不是产品价值本体
- 空间体验才是核心价值
- 偏离最小平台原则

---

## ✅ 正确的克制

1. **内容克制**：平台不绕过店主确认发布空间内容；AI 草稿只能作为可编辑、可丢弃、可确认的创作辅助
2. **计费克制**：平台不介入 Token 计费，店主自行承担
3. **社交克制**：不做无边界访客社交；玩家扮演 NPC 必须保持空间内、角色内、店主审批内
4. **地图克制**：不让地图表现成为产品价值本身
5. **UGC 克制**：只做平台基础设施和受控创作辅助，不替创作者最终发布内容

- v1.3 (2026-04-24): 放开店主审批的玩家扮演 NPC MVP，同时继续禁止跨空间社交网络
- v1.2 (2026-04-14): 按空间平台方向重写，替换旧不做清单
- v1.1 (2026-04-09): 按新主线重写，移除对旧地图主舞台的依赖口径
- v1.0 (2026-03-17): 初始版本，基于 AI-native 架构方向收束
