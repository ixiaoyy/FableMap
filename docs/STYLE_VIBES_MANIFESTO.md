# FableMap 视觉气质宣言 v0.1

## 目标

这份文档用于定义 FableMap 的“风格”不是简单滤镜，而是一套会同时影响：

- 色彩
- 光照
- 文案语气
- 音景建议
- 玩家情绪
- 地区可见性的世界镜头

它为未来的 Godot 呈现、Aesthetic Kernel 和 `region.vibe_profile` 提供统一边界。

## 核心立场

### 1. 风格不是贴图，而是视角

同一条街在不同 vibe 下，不只是“颜色变了”，而是世界解释方式变了。

例如：

- 在 `amber_evening` 下，街道像一段尚未说出口的私人回忆
- 在 `neon_nostalgia` 下，街道像消费城市对青春的持续回放
- 在 `gritty_cyberpunk` 下，街道像系统过热后的生存界面

### 2. 风格必须能反向约束生成

风格不是渲染层的附加效果，而应反向影响：

- `region.narrative_summary`
- `region.vibe_profile`
- `region.palette_hint`
- `poi.visual_hint`
- `state.active_lens`

### 3. 风格要允许温柔与锋利并存

FableMap 的气质不应只属于一种美术流派。

它既要容纳：

- 吉卜力式呼吸感
- 王家卫式潮湿与迟疑
- 脏冷赛博的压迫与霓虹残响
- 印象派式的光雾与轻盈

## 风格参数维度

每个 vibe 建议至少定义以下维度：

- **色温**：偏暖、偏冷、中性
- **饱和度**：低、中、高
- **对比度**：柔和、适中、强烈
- **雾感/空气感**：清澈、潮湿、朦胧、颗粒化
- **光源逻辑**：自然光、霓虹光、钠灯、室内溢出光
- **材质倾向**：木质、潮湿金属、旧墙面、玻璃反射、雨夜沥青
- **叙事语气**：温柔、暧昧、疲惫、锋利、神圣、荒凉
- **音景建议**：风、车流、广播、远处钟声、便利店门铃、潮湿电流噪声

## 当前建议风格簇

### 1. amber_evening

- 气质关键词：余温、迟暮、停留、未完成的告别
- 色温：暖
- 饱和度：中低
- 对比度：柔和
- 空气感：微潮、轻雾
- 适配场景：咖啡馆、住宅街角、黄昏通勤路、安静商店
- 文案倾向：克制、亲密、轻微惆怅
- palette 提示：`coffee_and_rose`

### 2. neon_nostalgia

- 气质关键词：商业光晕、青春残影、晚归、都市包装感
- 色温：冷暖混合
- 饱和度：中高
- 对比度：适中偏强
- 空气感：玻璃反射、潮湿霓虹
- 适配场景：便利店、商业街、地铁口、夜间步行道
- 文案倾向：热闹表面下的空心感
- palette 提示：`soda_neon`

### 3. ghibli_town

- 气质关键词：呼吸感、风、烟火、日常奇迹
- 色温：暖中带自然绿
- 饱和度：中
- 对比度：柔和
- 空气感：通透、轻风、云层移动感
- 适配场景：公园、居民区、小店、坡道、学校周边
- 文案倾向：温柔、好奇、略带童话但不幼稚
- palette 提示：`moss_and_bread`

### 4. gritty_cyberpunk

- 气质关键词：系统负荷、过热、压迫、生存界面
- 色温：冷
- 饱和度：局部高、整体低
- 对比度：强
- 空气感：颗粒、潮湿、电子噪声
- 适配场景：高架桥下、工业区、金融区边缘、深夜交通节点
- 文案倾向：锋利、疲惫、带有结构性不安
- palette 提示：`rust_and_laser`

### 5. impression_garden

- 气质关键词：光雾、柔焦、季节流动、短暂抚慰
- 色温：中性偏暖
- 饱和度：中低
- 对比度：低
- 空气感：柔焦、花粉、光斑
- 适配场景：花店、公园、水边步道、治愈型收集点
- 文案倾向：轻盈、抚慰、偏内心独白
- palette 提示：`petal_mist`

### 6. dark_gothic

- 气质关键词：神圣秩序、阴影、仪式、压抑庄严
- 色温：冷
- 饱和度：低
- 对比度：高
- 空气感：厚重阴影、烛光、石质回声
- 适配场景：政府建筑、教堂式机构、法院、纪念地标
- 文案倾向：庄严、审判感、带回声的权威叙事
- palette 提示：`ash_and_wine`

## 风格字段落点

建议在当前 Schema 中保持以下分工：

- `region.visual_style`：偏模板级视觉方案，供场景加载与大类 prefab 选择
- `region.vibe_profile`：偏镜头级审美气质，决定观看方式
- `region.palette_hint`：偏色板提示，供材质、UI、后处理使用
- `state.active_lens`：玩家当前主动切换的风格镜头
- `poi.visual_hint`：地点局部的材质、色调和装饰倾向

## Aesthetic Kernel 最小原则

未来即使只做一个很轻的可视化 demo，也应尽量支持：

- 同一份 `world.json`
- 在 2~3 种 vibe 下切换观看
- 让开发者立刻感知“同一街道，不同世界镜头”的反馈差异

这会极大提升原型阶段的可感知价值。

## 需要避免的坏味道

- 只做 LUT/滤镜，不改变叙事语气
- vibe 名称很美，但没有参数边界
- 一个区域同时叠太多风格，导致气质混乱
- 只追求流行视觉，不服务于地点语义

## 与未来系统的关系

- **文化诠释框架** 决定地点为什么这样被看见
- **Street as a Sentence** 决定整条街应共享怎样的统一镜头
- **Ethereal Radio** 可把 vibe 继续扩展到音景层
- **Godot 消费层** 负责把这些参数真正变成视觉和交互反馈
