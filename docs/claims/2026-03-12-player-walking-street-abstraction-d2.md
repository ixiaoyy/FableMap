# 认领说明：D2 · 像素角色行走与街道通行抽象

## 任务背景
目前 Web-2D 主舞台已经实现了地图渲染、交互缩放、动态扰动与共创层。为了进一步提升“世界”的可探索性，需要引入“玩家实体”。

本任务旨在 Web-2D 地图上添加一个受玩家控制的像素角色，并建立基于街道骨架的通行抽象，让地图从“可看”向“可行走”演进。

## 任务目标
1. **玩家实体渲染**：在 SVG 地图上添加一个代表玩家的像素/图标实体。
2. **行走逻辑**：支持键盘（WASD/方向键）控制玩家移动。
3. **街道通行抽象**：
    - 玩家倾向于在街道（roads）上移动。
    - 实现基础的碰撞或范围约束，防止玩家随意穿墙（非道路区域）。
    - 街道作为“移动通道”的语义抽象。
4. **相机跟随**：地图视口随玩家移动而自动偏移，保持玩家在中心。

## 不改边界
- 不引入复杂的物理引擎，保持基于 SVG 坐标的轻量实现。
- 暂时不实现复杂的 NPC 碰撞或多人同步。
- 移动动画保持基础的 CSS 或 transform 切换。
