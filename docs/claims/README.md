# FableMap 模块认领说明模板

## 目标

`docs/claims/` 用于记录“准备开发说明”或“模块认领说明”。

它的作用是：

- 防止多人重复开发同一个模块
- 提前声明修改边界
- 让协作者和 AI 都知道当前任务的上下文

## 什么时候必须提交

以下情况应先提交认领说明，再开始开发：

- 协议变更
- 新模块开发
- 跨文件功能开发
- H3 区域内容认领
- 其他可能和多人协作边界冲突的改动

以下情况可豁免：

- typo
- 纯格式调整
- 小范围文档润色
- 极小测试修正

## 推荐命名

建议文件名：

- `<date>-<module-name>.md`
- `<date>-<region-name>-<h3>.md`

例如：

- `2026-03-10-overpass-loader.md`
- `2026-03-10-shibuya-8a2a1072b59ffff.md`

## 模板

复制下面这份模板新建你的认领说明：

```md
# 模块认领说明

- 模块名 / 区域名：
- 负责人：
- 改动类型：协议 / 功能 / 区域内容 / 工具 / 文档 / 测试
- 当前状态：planned / in_progress / blocked / done

## 目标

本次准备完成什么：

## 计划修改范围

- 会改哪些文件或目录：

## 明确不改范围

- 哪些文件、模块或协议边界不在本次范围内：

## 依据的协议文档

- README.md
- CONTRIBUTING.md
- 其他相关协议文档：

## 预期产出

- 文档 / 代码 / 配置 / 测试 / 示例数据：

## 验证方式

- 计划执行哪些验证：

## 风险与备注

- 可能的冲突点、依赖和阻塞项：
```

## 注意

模块认领说明是“开工前声明”，**不能替代每次 commit 的文档说明**。

正确顺序应是：

1. 先认领
2. 再开发
3. 每次 commit 继续补本次改动说明
4. 提 PR 时引用认领说明和 commit 说明
