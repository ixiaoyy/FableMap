## 🔄 文档更新规则

1. **主线文档变更**：优先同步 [README.md](README.md)、[docs/PRODUCT_BRIEF.md](docs/PRODUCT_BRIEF.md)、[docs/ARCHITECTURE.md](docs/ARCHITECTURE.md)、[docs/CURRENT_TASKS.md](docs/CURRENT_TASKS.md)、[docs/AI_SHARED_TASKLIST.md](docs/AI_SHARED_TASKLIST.md)
2. **协议文档变更**：需要同步检查 [docs/PLAYER_STATE.md](docs/PLAYER_STATE.md)、[docs/WORLD_WRITEBACK_PROTOCOL.md](docs/WORLD_WRITEBACK_PROTOCOL.md)、[docs/WORLD_ORCHESTRATOR_PROTOCOL.md](docs/WORLD_ORCHESTRATOR_PROTOCOL.md) 是否仍与主链路一致
3. **历史文档处理**：不急于移动文件，优先在文档正文和索引中标明“历史参考”定位
4. **新增文档**：必须在本索引中注册，并标明它属于当前主线、核心协议、实施参考、历史参考还是协作留痕
5. **长期方向文档**：如果涉及长期愿景，必须明确说明它不会直接覆盖当前主链路优先级
