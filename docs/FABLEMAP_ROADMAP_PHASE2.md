# FableMap 新功能方向头脑风暴

## 背景

项目 Phase 1-11 + GC 群聊均已完成，核心产品框架基本成型。本文档梳理下一阶段可能的创新方向，供后续讨论和筛选。

---

## 方向 A：空间运营工具增强

**核心目标**：帮助店主更好地运营空间，提升数据可见性

### A1：空间数据看板

| 字段 | 描述 |
|------|------|
| 功能 | 访客活跃度、对话热度时段、最受欢迎角色 |
| 价值 | 帮助店主了解空间运营状况 |
| 复杂度 | 中 |
| 优先级 | P1 |

**候选指标**：
- 日/周/月访客量趋势
- 平均对话轮次
- 最活跃时间段
- NPC 互动排行
- 访客留存率（回访比例）

### A2：自动生成空间简介

| 字段 | 描述 |
|------|------|
| 功能 | 用 AI 根据角色和世界书生成空间宣传文案 |
| 价值 | 降低店主创作门槛 |
| 复杂度 | 高 |
| 优先级 | P2 |

**实现思路**：
- 读取空间角色卡 + 世界书 + 场景描述
- 调用 LLM 生成 2-3 句吸引人的简介
- 店主可编辑后保存

### A3：批量 NPC 生成

| 字段 | 描述 |
|------|------|
| 功能 | 店主批量创建配角 NPC（背景人物） |
| 价值 | 丰富空间氛围，降低创建成本 |
| 复杂度 | 中 |
| 优先级 | P2 |

**候选功能**：
- 批量导入角色卡（支持 JSON 数组）
- 一键生成背景 NPC（基于空间风格）
- NPC 关系图谱（NPC 之间的关系）

## 方向 B：访客体验深化

**核心目标**：让访客有更强的参与感和成长感

### B1：空间任务系统

| 字段 | 描述 |
|------|------|
| 功能 | 访客可接受 NPC 发布的任务/挑战 |
| 价值 | 增加探索动力和互动深度 |
| 复杂度 | 高 |
| 优先级 | P1 |

**候选任务类型**：

| 类型 | 描述 | 示例 |
|------|------|------|
| 探索任务 | 发现并拜访特定地点 | "拜访 3 家咖啡店" |
| NPC 任务 | 与特定 NPC 互动 | "和老王聊一次天" |
| 收集任务 | 收集特定角色/物品 | "收集 10 种不同表情" |
| 创作任务 | 创建新内容 | "创建一个自己的空间" |

**任务结构草案**：
```typescript
interface Quest {
  id: string
  tavern_id: string
  character_id?: string  // 可选，特定 NPC 发布
  title: string
  description: string
  objectives: QuestObjective[]
  rewards: QuestReward[]
  status: 'active' | 'completed' | 'failed'
}
```

### B2：关系进度可视化

| 字段 | 描述 |
|------|------|
| 功能 | 访客与 NPC 的关系以进度条/等级展示 |
| 价值 | 增加回访动力和游戏感 |
| 复杂度 | 低 |
| 优先级 | P1 |

**候选设计**：
- 关系等级：陌生人 → 认识 → 熟人 → 朋友 → 挚友
- 进度条显示距离下一等级还需要多少次互动
- 等级提升时显示动画/特效

### B3：访客成就墙

| 字段 | 描述 |
|------|------|
| 功能 | 解锁的对话里程碑、特殊称号 |
| 价值 | 增加成就感和分享素材 |
| 复杂度 | 中 |
| 优先级 | P2 |

**候选成就**：
- "第一位访客" - 成为某空间的第一个访客
- "话痨" - 单次对话超过 50 轮
- "常客" - 访问同一空间 10 次
- "收藏家" - 收集 20 种 NPC 表情
- "深夜访客" - 凌晨 2-4 点在便利店空间聊天

### B4：收藏空间

| 字段 | 描述 |
|------|------|
| 功能 | 访客收藏喜欢的空间，方便回访 |
| 价值 | 提升回访率 |
| 复杂度 | 低 |
| 优先级 | P2 |

## 方向 C：创作者生态

**核心目标**：帮助创作者更好地创作和传播

### C1：角色卡市场

| 字段 | 描述 |
|------|------|
| 功能 | 用户可分享/购买精选角色卡 |
| 价值 | 降低创作门槛，形成创作者经济 |
| 复杂度 | 高 |
| 优先级 | P1 |

**候选功能**：
- 角色卡上传与审核
- 标签分类（性格、风格、题材）
- 评分与评论
- 免费/付费角色卡
- 创作者分成（远期）

**版权风险提示**：
- 需要审核机制避免侵权角色卡
- 鼓励原创角色卡

### C2：空间主题包

| 字段 | 描述 |
|------|------|
| 功能 | 成套模板（学校线/便利店线/异世界线） |
| 价值 | 提供完整的空间搭建方案 |
| 复杂度 | 中 |
| 优先级 | P2 |

**候选主题包**：

| 主题 | 包含内容 |
|------|---------|
| 学校线 | 角色（老师、学生、保安）+ 玩法模板 + 世界书 |
| 便利店线 | 角色（店员、常客、老板）+ 场景描述 + 玩法模板 |
| 异世界线 | 角色（勇者、商人、公主）+ 世界书 + 背景设定 |

### C3：创作者排行榜

| 字段 | 描述 |
|------|------|
| 功能 | 按访客量/评分展示优秀空间 |
| 价值 | 激励创作者， 帮助发现优质内容 |
| 复杂度 | 低 |
| 优先级 | P2 |

**排行维度**：
- 访客量日榜/周榜/月榜
- 综合评分
- 新空间潜力榜

## 方向 D：社交与事件

**核心目标**：增强空间内外的社交互动

### D1：空间事件日历

| 字段 | 描述 |
|------|------|
| 功能 | 店主发布定时活动（节日限定 NPC） |
| 价值 | 制造稀缺感，增加回访动力 |
| 复杂度 | 中 |
| 优先级 | P1 |

**候选事件类型**：

| 类型 | 描述 | 示例 |
|------|------|------|
| 节日活动 | 节日期间限定角色/对话 | 春节 NPC、中秋嫦娥 |
| 限时剧情 | 限定时间内可触发的特殊剧情 | "今夜流星雨" |
| 联动活动 | 多空间联动叙事 | "寻找失踪的宝物" |

**技术要点**：
- 事件定时发布（cron job）
- 事件结束后的角色/对话清理
- 事件预告通知

### D2：访客留言板

| 字段 | 描述 |
|------|------|
| 功能 | 访客离开时给 NPC 留悄悄话 |
| 价值 | 增加访客间间接互动 |
| 复杂度 | 低 |
| 优先级 | P2 |

**候选功能**：
- 公开留言（所有访客可见）
- 私密留言（仅 NPC/店主可见）
- 留言点赞
- 热门留言展示

### D3：跨空间剧情线

| 字段 | 描述 |
|------|------|
| 功能 | 多空间联动叙事（需主人授权） |
| 价值 | 创造更大的故事世界 |
| 复杂度 | 高 |
| 优先级 | P3 |

**候选设计**：
- 跨空间任务（需要拜访多个空间完成）
- 空间间 NPC 联动对话
- 主线剧情推进

## 功能优先级汇总

根据价值（用户需求）和实现复杂度排序：

| 优先级 | 功能 | 方向 | 原因 |
|--------|------|------|------|
| P1 | **关系进度可视化** | B | 低复杂度，高用户价值 |
| P1 | **空间数据看板** | A | 高价值，中等复杂度 |
| P1 | **空间任务系统** | B | 高价值，高价值感 |
| P1 | **空间事件日历** | D | 制造稀缺感，提升回访 |
| P1 | **角色卡市场** | C | 创作者经济基础设施 |
| P2 | **访客成就墙** | B | 增加游戏感 |
| P2 | **访客留言板** | D | 低复杂度，增加互动 |
| P2 | **空间主题包** | C | 提供完整解决方案 |
| P2 | **收藏空间** | B | 提升回访率 |
| P2 | **批量 NPC 生成** | A | 降低创建成本 |
| P3 | **跨空间剧情线** | D | 高复杂度，需长期规划 |
| P3 | **自动生成空间简介** | A | 技术难度高，效果不确定 |
| P3 | **数据分析管道** | E | 基础设施，长期价值 |
| P3 | **AB 测试框架** | E | 基础设施 |

## 待讨论问题

1. **核心方向**：是继续深化空间体验（P1 功能），还是拓展生态（角色卡市场）？

2. **任务系统范围**：任务由谁创建？
   - 选项 A：店主创建
   - 选项 B：平台预设 + 店主自定义
   - 选项 C：AI 自动生成

3. **社交边界**：
   - WHAT_NOT_TO_BUILD 明确不做访客间实时社交
   - 留言板/排行榜是否符合这个边界？

4. **付费模式**：
   - 角色卡市场是否需要付费功能？
   - 如果需要，如何设计分成机制？
